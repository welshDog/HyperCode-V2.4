## 🎯 Quick Start

### Prerequisites
```bash
# Required software
- Python 3.11+
- PostgreSQL 15+
- Redis 7+
- Node.js 18+ (for dashboard)
- Docker & Docker Compose

# API Keys needed
- Discord Bot Token
- MintMe API Key (if available)
- OpenAI API Key (optional, for advanced NLP)
```

```bash

cp .env.example .env
nano .env  # Add your API keys

make test-community-features

# 🎉 System ready!
# - Bot: Running in Discord
# - Dashboard: http://localhost:5000
# - API: http://localhost:8000/api
# - Metrics: http://localhost:9090
```

```
        ↓

        ↓

        ↓

        ↓
```

## 🎨 Feature Implementation

### Feature 1: AI Contribution Classifier

**File**: `src/ai/contribution_classifier.py`

```python
# Usage in bot
from src.ai.contribution_classifier import AIContributionClassifier

@bot.event
async def on_message(message):
    if message.author.bot:
        return
    
    async with db.session() as session:
        classifier = AIContributionClassifier(session)
        
        # Analyze message
        score = await classifier.classify_message(message)
        
        if score:
            # Process contribution
            community_service = CommunityService(session)
            await community_service.process_contribution(
                user_id=message.author.id,
                message_id=message.id,
                channel_id=message.channel.id,
                guild_id=message.guild.id,
                score=score,
            )
            
            # Send notification
            await message.add_reaction("✅")
            await message.author.send(
                f"🎉 You earned **{score.final_reward}** BROski$ for "
                f"{score.contribution_type.value.replace('_', ' ')}!\n"
                f"Reason: {score.reasoning}"
            )
```

### Feature 3: MintMe Blockchain Integration

**File**: `src/integrations/mintme/api_client.py`

```python
# Convert internal tokens to blockchain
async def convert_user_tokens(user_id: int, amount: int):
    async with MintMeAPIClient() as client:
        # Get user's wallet address
        wallet = await get_user_wallet(user_id)
        
        # Calculate MintMe tokens
        mintme_tokens = amount / 1000
        
        # Transfer to blockchain
        tx_hash = await client.transfer_tokens(
            to_address=wallet,
            amount=mintme_tokens,
            token_symbol="BROski",
        )
        
        if tx_hash:
            # Record transaction
            await record_blockchain_transfer(
                user_id=user_id,
                amount=amount,
                tx_hash=tx_hash,
            )
            
            return tx_hash
        else:
            raise Exception("Blockchain transfer failed")
```

**Automatic Conversion**:

```python
# Auto-convert when threshold reached
```

## 🪙 Token Integration

### Internal Token System

**BROski$** (Internal Discord tokens):
- Earned through contributions
- Stored in PostgreSQL
- Instant distribution
- No gas fees
- Used for Discord perks

### Blockchain Token System

**BROski** (MintMe blockchain):
- Real cryptocurrency
- 1000 BROski$ = 1 BROski token
- Requires wallet address
- Small gas fees
- Transferable, tradable

### Conversion Process

```python
# User initiates conversion
@bot.command(name="convert")
async def convert_tokens(ctx, amount: int):
    """Convert internal tokens to blockchain."""
    
    # Validate amount
    if amount < 1000:
        await ctx.send("❌ Minimum conversion is 1000 BROski$")
        return
    
    # Check balance
    balance = await get_user_balance(ctx.author.id)
    if balance < amount:
        await ctx.send(f"❌ Insufficient balance. You have {balance} BROski$")
        return
    
    # Request wallet address if not set
    wallet = await get_user_wallet(ctx.author.id)
    if not wallet:
        await ctx.send(
            "⚠️ No wallet address on file. "
            "Please set your MintMe wallet address with `/setwallet <address>`"
        )
        return
    
    # Confirm conversion
    mintme_tokens = amount / 1000
    embed = discord.Embed(
        title="🔄 Token Conversion",
        description=f"Convert **{amount:,}** BROski$ to **{mintme_tokens:.2f}** BROski tokens?",
        color=discord.Color.blue(),
    )
    embed.add_field(name="Your Wallet", value=wallet)
    embed.add_field(name="Est. Gas Fee", value="~0.001 MINTME")
    
    view = ConfirmationView()
    message = await ctx.send(embed=embed, view=view)
    
    # Wait for confirmation
    await view.wait()
    
    if view.confirmed:
        # Process conversion
        try:
            tx_hash = await convert_user_tokens(ctx.author.id, amount)
            
            await message.edit(
                content=f"✅ Conversion successful!\n"
                        f"Transaction: `{tx_hash}`\n"
                        f"Your BROski tokens will arrive in ~5-10 minutes.",
                view=None,
            )
        except Exception as e:
            await message.edit(
                content=f"❌ Conversion failed: {str(e)}",
                view=None,
            )
    else:
        await message.edit(content="❌ Conversion cancelled", view=None)
```

### Wallet Management

```python
@bot.command(name="setwallet")
async def set_wallet(ctx, wallet_address: str):
    """Set MintMe wallet address for conversions."""
    
    # Validate wallet format
    if not is_valid_wallet_address(wallet_address):
        await ctx.send("❌ Invalid wallet address format")

    await save_user_wallet(ctx.author.id, wallet_address)
    
    await ctx.send(
        f"✅ Wallet address set!\n"
        f"You can now convert BROski$ to blockchain tokens.\n"
        f"Current balance: {await get_user_balance(ctx.author.id):,} BROski$"
    )
```

## 🎓 Best Practices

### Token Distribution

1. **Start Conservative**: Begin with lower rewards, increase based on community response
2. **Monitor Inflation**: Track total supply and distribution velocity
3. **Prevent Gaming**: Use AI quality scores to prevent spam
4. **Transparent Rules**: Publish contribution → reward mapping
5. **Regular Audits**: Review distribution patterns monthly

### Community Management

1. **Encourage Quality**: Higher rewards for detailed, helpful contributions
2. **Welcome Newcomers**: Extra rewards for helping new members
3. **Celebrate Milestones**: Special rewards at token thresholds
4. **Foster Engagement**: Daily challenges and events
5. **Be Responsive**: Address concerns promptly

### Technical Operations

1. **Database Optimization**: Regular index updates, query optimization
2. **Caching Strategy**: Cache frequently accessed data
3. **Gradual Rollouts**: Test with small user group first
4. **Version Control**: Tag all releases, maintain changelog
5. **Documentation**: Keep docs updated with each release
