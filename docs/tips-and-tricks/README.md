## 🎯 Philosophy

Every guide in this knowledge base follows these principles:

- **🟢🟡🔴 Color-coded risk levels** — Know what's safe before you try it
- **Step-by-step workflows** — No scattered instructions, just clear paths
- **Real examples** — From actual HyperCode commits, not abstract theory
- **Safety first** — Always show the safe way, warn about dangers
- **ADHD/Dyslexia friendly** — Chunked content, visual anchors, emojis
- **5-10 minute reads** — Get in, learn, get out

## 🤝 Contributing to Tips & Tricks

**Want to add a guide?** Here's the template:

### Guide Structure
1. **Title + Metadata** — What you'll learn, reading time, skill level
2. **Problem Statement** — What issue does this solve?
3. **Step-by-Step Solution** — Clear, numbered steps
4. **Risk Indicators** — 🟢🟡🔴 for every operation
5. **Real Examples** — From HyperCode if possible
6. **Troubleshooting** — Common issues + fixes
7. **Related Guides** — Links to other tips

### Formatting Rules
- Max 3 sentences per paragraph
- Use emojis as visual anchors
- Code blocks for all commands
- Risk level before every dangerous operation
- Keep guides under 10 minutes reading time

### Submit Your Guide
1. Create your markdown file in `docs/tips-and-tricks/`
2. Follow the naming convention: `lowercase-with-dashes.md`
3. Add it to this index (in the right category)
4. Submit a pull request
5. Tag it with `documentation` label
